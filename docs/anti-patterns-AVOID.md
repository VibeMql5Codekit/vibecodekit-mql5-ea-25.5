# Anti-patterns to AVOID

> **Honest disclaimer.** The architectural anti-patterns below (and the
> `AP-1…AP-25` regex/AST detectors that operationalise them) are
> **project-defined heuristics** designed by this kit. They are
> opinionated guardrails — not industry standards, not certifications,
> and not substitutes for code review or live-account validation. The
> `> 6 inputs` threshold (AP-5), the `15/17` Trader-17 gate, and the
> `48/64` matrix threshold (TEAM mode) are project choices, not
> universal truths.

This document lists 5 anti-patterns identified in the parent `vibecodekit-hybrid-ultra`
(VCK-HU) repository that vibecodekit-mql5-ea **MUST NOT** inherit. These were
identified via self-audit before finalizing Plan v5.

Every Devin session implementing this kit must read this document at start
to avoid drift.

---

## 1. Over-engineering

**VCK-HU symptom:** 42 slash commands, 7 sub-agents, 6-layer permission, 96-probe
audit, 33 hook events, 3-tier memory, 3 MCP servers, RRI 5 personas × 25 questions
× 3 modes = 375 questions. Total LOC ~5000-7000 Python.

**v5 mitigation:**
- Each command must pass gating: "EA dev không có cái này thì work-flow break ở đâu?"
- 30 commands (NO master `/mql5` router)
- 8 sub-agents only loaded for ENTERPRISE mode (default PERSONAL skips)
- Each Python script < 200 LOC + 1 single responsibility (no god module)
- 7-layer permission only ENTERPRISE mode mandatory; PERSONAL uses 2 layers
- 64-cell quality matrix only ENTERPRISE; PERSONAL uses 16 cells

**Verdict for v5:** PARTIAL — comprehensive but each feature gating-justified.

---

## 2. Conformance audit self-referential

**VCK-HU symptom:** 96 probes test "did Python module load?" not "did kit produce
working artifact?". Probe `tests/test_docs_count_sync.py` verifies version mirrors
generated by the tool itself — circular.

**v5 mitigation:**
- 70 conformance tests = 10 e2e EXTERNAL (real MetaEditor compile + Strategy Tester)
  + 60 internal probes (module load, schema validation)
- Every e2e test MUST run actual MQL5 compilation OR actual tester run
- Internal probes only validate config/schema, NOT functionality

**Verdict for v5:** PARTIAL — hybrid approach, e2e dominates for functional gates.

---

## 3. God Module

**VCK-HU symptom:** `subagent_runtime.py` (355 LOC) handles 7 profiles + ACL +
audit log + file mutation gate. `permission_engine.py` stacks 6 layers. Master
`/vibe` command routes 42 commands.

**v5 mitigation:**
- Each Python script < 200 LOC strict ceiling
- Each module has 1 responsibility (no fan-in/fan-out aggregation)
- 7-layer permission split into 7 separate scripts (`layer1_*.py` ... `layer7_*.py`)
- NO master `/mql5` command — user invokes individual commands

**Verdict for v5:** NO — strict adherence enforced via `audit-plan-v5.py` post-phase
check.

---

## 4. Dead code (`query_loop` + `tool_executor`)

**VCK-HU symptom:**
- `query_loop.py` (244 LOC) — generic plan executor (`run_plan`, `_classify_error`,
  `_dispatch_recovery`)
- `tool_executor.py` (587 LOC) — 16 generic tool wrapper functions (file_read,
  shell_run, etc.)
- Both are "legacy" but still present; `/vibe-run <plan>` mentioned in commands
  but not integrated into CI.

**v5 mitigation:**
- DO NOT port `query_loop.py` — Devin sessions execute plans natively
- DO NOT port `tool_executor.py` — each command is a standalone Python script
- Per-phase implementation via direct command CLI invocation, no plan executor

**Verdict for v5:** NO — explicitly forbidden by audit; never present.

---

## 5. Intent router

**VCK-HU symptom:**
- `intent_router.py` (683 LOC) + `pipeline_router.py` (273 LOC) = 956 LOC
- Master `/vibe` command → `intent_classifier.py` → 25 sub-dispatch rules → actual
  `/vibe-blueprint` call hidden 4 layers deep
- LLM-primary routing with Python fallback adds latency and complexity

**v5 mitigation:**
- DO NOT port `intent_router.py` or `pipeline_router.py`
- DO NOT add master `/mql5` command — user invokes `/mql5-build`, `/mql5-lint`,
  `/mql5-compile`, etc. DIRECTLY
- Discoverability via `/mql5-doctor` (lists installed commands) + COMMANDS.md
  reference doc, NOT via routing

**Verdict for v5:** NO — explicitly forbidden by audit; never present.

## Enforcement

`scripts/audit-plan-v5.py` runs PRE and POST each phase to verify:

```python
FORBIDDEN_FILES = [
    "scripts/query_loop.py",
    "scripts/tool_executor.py",
    "scripts/intent_router.py",
    "scripts/pipeline_router.py",
    "scripts/master_command.py",
    "scripts/vibecodekit_mql5/query_loop.py",
    "scripts/vibecodekit_mql5/tool_executor.py",
    "scripts/vibecodekit_mql5/intent_router.py",
    "scripts/vibecodekit_mql5/pipeline_router.py",
]

FORBIDDEN_PATTERNS = [
    r"def run_plan\(",
    r"def classify_intent\(",
    r"class IntentRouter",
    r"class PipelineRouter",
    r"class ToolExecutor",
]
```

If audit detects any forbidden file or pattern, phase merge is blocked.

## When in doubt

**Test:** "EA dev không có feature X này thì work-flow break ở đâu cụ thể?"

If you can't answer with a specific user-facing breakage, drop the feature.

If you DO add a feature, document the answer in the PR description.
